## 2024-05-24 - Accessibility and Discoverability via Shortcuts and Tooltips
**Learning:** PyQt applications greatly benefit from standard keyboard accessibility practices like mnemonics (using `&` in labels/buttons) and `setBuddy` to link labels with their corresponding input fields, enabling users to jump to inputs using keyboard shortcuts (e.g., `Alt+C`). Adding explicit tooltips on interactive elements provides helpful contextual clues without cluttering the UI. Minor mnemonic overlaps (e.g., `Alt+C` for two different actions) are handled gracefully by Qt via focus cycling, but should generally be minimized where possible.
**Action:** Always check for missing mnemonics in desktop application labels/buttons and implement `setBuddy` when adding new inputs. Ensure all primary interactive controls have descriptive tooltips to aid usability.

## 2026-07-03 - Explicit Feedback in Long-Running Actions and Error States
**Learning:** Users often miss subtle text added to a read-only log box. For important error states (like missing input files before starting a process), an explicit `QMessageBox` alert provides an unavoidable and clear notification. Furthermore, for long-running asynchronous tasks (like image stitching), simply disabling a button isn't enough; changing its text (e.g., from "Start Processing" to "Processing...") provides crucial visual feedback that the application hasn't frozen and work is underway. Finally, providing empty states (like `setPlaceholderText` on text areas) guides the user's expectations.
**Action:** When designing UI flows that require user input before proceeding, use modal alerts or inline validation instead of silent logging. Always provide visual state changes (like button text or spinners) for operations that take more than a few hundred milliseconds. Ensure empty containers have placeholder text explaining their purpose.
