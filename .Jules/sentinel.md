## 2024-07-04 - HTML Injection via PyQt5 QTextEdit
**Vulnerability:** The application was vulnerable to HTML injection via the GUI's log text box. PyQt5's `QTextEdit.append()` method interprets input as rich text (HTML) by default. Untrusted input, such as malicious file paths, passed directly to `append()` could be executed or incorrectly formatted by the Qt rich text engine.
**Learning:** `QTextEdit` and similar Qt widgets default to rich text rendering for convenience. Passing unescaped strings directly to these widgets creates an injection vector, particularly when dealing with file names or other external input. When mitigating, `html.escape()` must be used. Furthermore, wrapping the escaped string in a basic HTML tag like `<span>` ensures that Qt parses it as rich text and decodes the entities properly. Failing to wrap it can lead to Qt displaying the literal `&lt;` entities.
**Prevention:** Always sanitize untrusted input using `html.escape()` before displaying it in Qt rich text components like `QTextEdit` or `QLabel` (when rich text is enabled). When appending escaped text to force rich text parsing, encapsulate it in a standard HTML tag.

## 2024-07-05 - Insecure Local Configuration Storage & Missing Validation
**Vulnerability:** The application was vulnerable to Insecure File Storage (CWD modification) and potential local DoS due to missing type validation. It saved `calibration.json` in the Current Working Directory (CWD) and blindly trusted its data types. If run in a shared/temp directory, an attacker could symlink this file to overwrite sensitive files when the app saves. Additionally, injecting non-float types (e.g., strings/arrays) could cause unhandled exceptions during mathematical operations in the application.
**Learning:** Storing configuration files in the CWD without an explicit, predictable, and secure path creates a risk when the application is executed from untrusted directories. Furthermore, external configuration files (even local ones) must be treated as untrusted input and type-validated when parsed, preventing unexpected exceptions and crashes.
**Prevention:** Always use predictable, user-specific locations for application data (e.g., `os.path.expanduser("~/.config_name.json")`). Always validate the types and bounds of data loaded from external sources (JSON, YAML, etc.) before assigning them to application state variables.
