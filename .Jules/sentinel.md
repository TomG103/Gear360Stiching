## 2025-05-18 - Insecure Local Configuration File Path and Missing Type Validation

**Vulnerability:** The application was loading and saving its calibration state (`calibration.json`) using a relative path, which resolves to the current working directory. Additionally, values read from this JSON file for `yaw2`, `pitch2`, and `roll2` were not explicitly type-checked or converted to numeric types before being used in trigonometric mathematical operations for image stitching.

**Learning:** Using the current working directory for configuration files exposes the application to local symlink attacks and unintended file overwrites, particularly if the tool is run from various directories or via shortcuts. Failing to cast externally loaded JSON values introduces a risk of local Denial of Service (DoS); an attacker or malformed configuration could inject strings into fields expected to be floats, causing the application to crash with an unhandled `TypeError` when evaluating the values with `np.cos` and `np.sin`.

**Prevention:** Always use explicit, user-specific absolute paths for local application state or configuration files (e.g., `os.path.expanduser("~/.config_filename.json")`). Always explicitly validate and cast the data types of values read from external sources (such as JSON configuration files) before utilizing them in operations that expect specific types.
