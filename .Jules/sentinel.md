## 2024-07-04 - HTML Injection via PyQt5 QTextEdit
**Vulnerability:** The application was vulnerable to HTML injection via the GUI's log text box. PyQt5's `QTextEdit.append()` method interprets input as rich text (HTML) by default. Untrusted input, such as malicious file paths, passed directly to `append()` could be executed or incorrectly formatted by the Qt rich text engine.
**Learning:** `QTextEdit` and similar Qt widgets default to rich text rendering for convenience. Passing unescaped strings directly to these widgets creates an injection vector, particularly when dealing with file names or other external input. When mitigating, `html.escape()` must be used. Furthermore, wrapping the escaped string in a basic HTML tag like `<span>` ensures that Qt parses it as rich text and decodes the entities properly. Failing to wrap it can lead to Qt displaying the literal `&lt;` entities.
**Prevention:** Always sanitize untrusted input using `html.escape()` before displaying it in Qt rich text components like `QTextEdit` or `QLabel` (when rich text is enabled). When appending escaped text to force rich text parsing, encapsulate it in a standard HTML tag.
